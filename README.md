# Агент HTML-first презентаций

Этот репозиторий содержит многоагентную систему для создания презентаций из проверенного HTML-исходника с экспортом в PNG, PDF, редактируемый PPTX и безопасные SVG-ассеты.

Активный процесс строится вокруг навыка: `$presentation-design` — единственный презентационный навык, а полное создание колоды должно проходить через настроенных рабочих субагентов.

## Что создаётся

Каждая презентация живёт в самодостаточной папке:

```text
presentations/<slug>/
├── package.json
├── asset-manifest.json
├── assets/
├── slides/
├── scripts/
└── exports/
```

HTML является источником истины. Статические экспорты создаются локальными скриптами, скопированными в папку конкретной презентации.

## Агентский процесс

Полное создание использует шесть рабочих агентов:

| Рабочий агент | Ответственность |
|---|---|
| `presentation-researcher` | Сюжет, факты, план слайдов, заглушки и замысел заметок докладчика |
| `presentation-asset-builder` | Брендовые и содержательные материалы, манифест защищённых визуалов, SVG и схемы |
| `presentation-direction-designer` | Двухслайдовое направление: обложка и плотный содержательный слайд |
| `presentation-html-builder` | Итоговый HTML-исходник, общие CSS/компоненты и исправления после проверки |
| `presentation-qa-reviewer` | Проверка в браузере, свежий PNG-экспорт и последовательная проверка слайдов |
| `presentation-exporter` | Финальные PDF, PNG, редактируемый PPTX и безопасные SVG-артефакты |

Главный агент остаётся выпускающим редактором: держит бриф, принимает сюжетные решения, выдаёт ограниченные задания, направляет исправления после проверки и сообщает о финальных артефактах. Во время полного процесса он не обходит обязательные рабочие фазы.

## Как запускать

Явно используйте общий навык:

```text
Используй $presentation-design, чтобы создать 12-слайдовую презентацию о продуктовой стратегии. Нужны HTML, PDF, PNG и редактируемый PPTX.
```

Навык выполнит следующие шаги:

1. Уточнит бриф, целевые форматы, ограничения и недостающие входные данные.
2. Создаст `presentations/<slug>/`.
3. Скопирует экспортные скрипты навыка в эту папку.
4. Соберёт или запросит нужные материалы и `asset-manifest.json`.
5. Подготовит двухслайдовое визуальное направление.
6. Соберёт полную HTML-презентацию.
7. Экспортирует свежие PNG и проверит каждый слайд по порядку.
8. Создаст запрошенные финальные форматы только после чистой проверки.

## Команды экспорта

Запускайте команды из папки конкретной презентации:

```bash
node scripts/export_deck_png.mjs --slides slides --out exports/png
node scripts/export_deck_pdf.mjs --slides slides --out exports/deck.pdf
node scripts/export_deck_pptx.mjs --slides slides --out exports/deck.pptx
```

Для однофайловой deck-stage презентации:

```bash
node scripts/export_deck_stage_png.mjs --html deck.html --out exports/png --slides 12
node scripts/export_deck_stage_pdf.mjs --html deck.html --out exports/deck.pdf
```

Не запускайте экспортные скрипты из корня репозитория. Папка презентации должна быть самодостаточной.

## Контроль качества

Финальная сдача требует:

- наличия `asset-manifest.json`;
- защищённых визуалов, закрытых реальными PNG/JPG/WebP или явно заблокированных;
- свежего PNG-экспорта каждого слайда;
- `exports/slide-qa.md` с одной строкой на каждый слайд;
- чистого прохода `1..N` после последнего изменения исходников;
- запрошенных PDF/PPTX/PNG/SVG-артефактов в `presentations/<slug>/exports/`.

Редактируемый PPTX использует ограниченный HTML. Если нужен PPTX, следуйте `.agents/skills/presentation-design/references/pptx-authoring.md` с самого начала.

## Настройка рантаймов

OpenCode:

- агенты: `.opencode/agents/presentation-*.md`
- инструменты: `.opencode/tools/presentation-export.ts`
- разрешение навыка: `opencode.json`

Codex:

- агенты: `.codex/agents/presentation-*.toml`
- конфигурация: `.codex/config.toml`

Общий навык:

- `.agents/skills/presentation-design/SKILL.md`
- `.agents/skills/presentation-design/references/`
- `.agents/skills/presentation-design/scripts/`
- `.agents/skills/presentation-design/assets/`

При изменении поведения рабочих агентов одновременно обновляйте оба рантайма и справочники общего навыка.

Для smoke-проверки создайте `presentations/smoke-html-first/`, скопируйте скрипты навыка в `scripts/`, создайте пустой манифест `{"visuals":[]}` и запустите браузерную проверку, а затем PNG/PDF-экспорт из этой папки.
