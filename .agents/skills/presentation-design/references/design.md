# Дизайн презентации

Этот справочник задает базовые правила композиции. Он помогает быстро принять решения: что должно быть на слайде, как зритель будет читать материал и где нужны доказательства, а не украшения.

Подробные правила лежат отдельно:

- брендовые материалы и `brand-spec.md`: `brand-assets.md`;
- выбор визуального направления: `design-directions.md`;
- ограничения экспорта HTML/PPTX: `html-decks.md`, `pptx-authoring.md`.

## Быстрый порядок

1. Сначала выясни контекст: аудитория, цель, сюжет, формат сдачи, похожие презентации, бренд, данные, скриншоты, графики.
2. Если есть конкретный бренд или продукт, пройди строгую проверку материалов из `brand-assets.md`.
3. Если стиль не задан, предложи 3 разных направления из `design-directions.md`.
4. Собери проверку визуальной грамматики из 2 слайдов: обложка и плотный содержательный слайд.
5. После выбора зафиксируй дизайн-токены (`design tokens`): цвет, шкала шрифтов, сетка, обработка изображений, стиль графиков, при необходимости шкала эмоджи, футер, нумерация и заметки. Дизайн-токены — это список постоянных настроек, чтобы все слайды выглядели как одна система.

## Правила композиции

- Один слайд выполняет одну работу: ориентирует, доказывает, сравнивает, объясняет процесс, просит решение или фиксирует следующий шаг.
- Если на слайде есть доказательство или график, заголовок должен быть выводом, а не названием темы.
- Доказательство важнее декора: реальный скриншот, график, диаграмма, цитата, изображение продукта или примечание об источнике.
- Компоновка должна поддерживать сюжет. Если меняется только цвет, направление еще не выбрано.
- Похожие элементы должны иметь одинаковые размеры, отступы и выравнивание.
- Футер, номер страницы и примечания об источниках должны быть стабильными на всех слайдах.

## Роли слайдов

Чередуй роли, чтобы презентация не превратилась в один шаблон с новым текстом:

1. Обложка: задает тон, обещание и главный визуальный сигнал.
2. Повестка: показывает карту истории, а не обычный список пунктов.
3. Разделитель: дает ритм и отмечает смену темы.
4. Тезис: одна мысль и одно доказательство или пример.
5. Вывод из данных: график/таблица + заголовок-вывод + источник.
6. Сравнение: варианты, сравнимые по одной логике.
7. Процесс: 3-6 шагов с параллельными подписями.
8. Цитата или голос клиента: крупная цитата с автором и контекстом.
9. Дорожная карта: этапы, сроки, неопределенность и ответственные.
10. Финал и просьба: решение, ответственный, следующий шаг; не обычное «спасибо».

## Читаемость

Для `1920x1080`:

- заголовок: обычно `56-96px`;
- заголовок раздела: `72-120px`;
- основной текст: `24px` или больше;
- подписи и примечания об источниках: обычно `18px` или больше;
- подписи на графиках должны читаться в режиме показа;
- обычный содержательный слайд: максимум 3-5 визуальных групп.

Для `960x540pt` редактируемого PPTX:

- сохраняй иерархию в `pt`;
- не делай тесные текстовые блоки;
- держи безопасные поля;
- сложные эффекты заменяй простыми фигурами или слоями изображений, если текст должен остаться редактируемым.

## Учебная система эмоджи

На образовательных слайдах эмоджи можно использовать как учебные подсказки: маркеры шагов, сигналы "пример / практика / проверка", мнемоники и навигационные якоря. Эмодзи не заменяют диаграмму, график, скриншот, предметное изображение, учебную иллюстрацию или доказательство.

Правила:

- Сначала определи аудиторию и тон: дети, подростки, взрослые, корпоративное обучение. Для взрослых используй меньше эмоджи и более спокойную сетку.
- Для учебной темы сначала реши, нужен ли рисунок, фотография, карта, схема или другое предметное изображение. Если нужен, подбери материал и спланируй подписи до выбора эмодзи.
- Задай шкалу эмоджи в токенах, а не случайными inline-размерами.
- Используй 3-5 повторяемых ролей подсказок: понятие, пример, упражнение, предупреждение, повторение. Одна роль = один эмоджи и один диапазон размеров.
- Меняй размер осознанно: большой эмоджи для подсказки раздела или обложки, средний для карточки или шага, маленький для подписи в строке.
- Для `Young ESL` перед созданием SVG-ассета для простой подсказки, маркера или игровой роли сначала подбери системный эмоджи и размер `xs/sm/md/lg/xl` под конкретный слайд. К SVG переходи только если эмоджи не найден, плохо читается или может быть понят неправильно.
- Держи эмоджи в отдельной визуальной зоне: колонка маркеров, полоса упражнений или полоса повторения. Не вставляй их хаотично в заголовки.
- Если эмоджи только украшает, добавляй `aria-hidden="true"`. Если он несет смысл, повторяй этот смысл текстом на слайде или в заметках.
- В PPTX проверяй эмоджи глазами: читается ли знак, не сломал ли он интервал, не стал ли слишком крупным или мелким после экспорта.

Рекомендуемые CSS-токены для `1920x1080`:

```css
:root {
  --emoji-xs: 28px;
  --emoji-sm: 40px;
  --emoji-md: 64px;
  --emoji-lg: 96px;
  --emoji-xl: 136px;
}

.emoji {
  display: inline-flex;
  width: 1em;
  height: 1em;
  align-items: center;
  justify-content: center;
  font-family: "Apple Color Emoji", "Segoe UI Emoji", "Noto Color Emoji", sans-serif;
  line-height: 1;
}

.emoji[data-size="sm"] { font-size: var(--emoji-sm); }
.emoji[data-size="md"] { font-size: var(--emoji-md); }
.emoji[data-size="lg"] { font-size: var(--emoji-lg); }
```

## Анти-шаблонные правила

Избегай:

- generic AI glow: типового "сияния про AI";
- purple-blue gradients и floating spheres: фиолетово-синих градиентов и парящих шаров;
- случайных emoji-icons вне учебной системы подсказок;
- фальшивых панелей показателей, фальшивых графиков и выдуманных чисел;
- угаданных логотипов, скриншотов и брендовых цветов;
- одинакового card stack на каждом слайде;
- силуэтов из стоковых шаблонов и абстрактных технологичных фонов без доказательств.

Используй:

- реальные материалы и примечания об источниках;
- ясную типографическую иерархию;
- стабильную систему сетки и отступов;
- содержательные диаграммы и графики;
- честные заглушки с подписями;
- размерные эмоджи-подсказки для образовательных презентаций, когда они помогают учиться;
- одну сильную визуальную идею на ключевом слайде.

## Данные и утверждения

- Не выдумывай цифры, ссылки на источники, цитаты и статус релиза.
- Помечай демонстрационные данные прямо на слайде или в заметках.
- Таблицы используй для точного поиска значений, графики - для закономерностей.
- Подсвечивай одно число или одно изменение, которое аудитория должна запомнить.
- Источники держи рядом с графиком/таблицей или в заметках докладчика.

## CSS helpers

```css
* {
  box-sizing: border-box;
}

.slide {
  width: 1920px;
  height: 1080px;
  overflow: hidden;
}

.grid {
  display: grid;
  grid-template-columns: repeat(12, minmax(0, 1fr));
  gap: 24px;
}

p,
h1,
h2 {
  text-wrap: pretty;
}
```

Эффект должен создавать иерархию, доверие, ритм или ощущение материала. Если после удаления эффекта смысл слайда не меняется, эффект лишний.
