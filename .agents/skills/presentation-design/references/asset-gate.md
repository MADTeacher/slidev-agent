# Проверяемый asset gate

Этот файл - главный источник правил для предметных визуалов в презентациях. Он
используется до визуального дизайна, до двухслайдового направления, до массовой
сборки и перед экспортом.

## Принцип 0

Запись причины в `asset-notes.md` не является разрешением продолжать. Для
защищённого визуала нужен реальный PNG/JPG/WebP либо явное
подтверждение пользователя на резервный вариант.

## Защищённые визуалы

Защищённый визуал - это изображение, где зритель должен узнать конкретный
предметный образ:

- человек, ребенок, персонаж, часть тела;
- одежда, аксессуар, бытовая сцена;
- животное, еда, растение, транспорт, техника;
- предметная карточка для маленьких учеников;
- продукт, место, исторический артефакт, медицинский или учебный объект.

SVG, HTML/CSS, canvas, скриншот векторной имитации и SVG-origin PNG не закрывают
этот путь. Для стрелок, схем, выносок, декоративных маркеров, абстрактных
обложек и нативных SVG-слайдов этот gate не запрещает SVG, если визуал не является
защищённым.

## Обязательные файлы

В рабочей директории каждой презентации должен быть `asset-manifest.json`.
`asset-notes.md` можно вести для человека, но проверяемым источником истины
является manifest.

Минимальный manifest для презентации без предметных визуалов:

```json
{
  "target_format": "html",
  "visuals": []
}
```

Запись для защищённого визуала:

```json
{
  "id": "boy-winter",
  "slides": ["09-boy-winter.html"],
  "visual_type": "character",
  "protected": true,
  "role": "boy wearing winter clothes",
  "source_kind": "imagegen",
  "path": "assets/content/boy-winter.png",
  "source_url": null,
  "license": "generated",
  "evidence": "imagegen output saved locally",
  "created_with": "imagegen",
  "status": "ok",
  "fallback_approved_by_user": false,
  "fallback_reason": null
}
```

## Допустимые состояния

Защищённый визуал проходит проверку только при выполнении всех условий:

- `source_kind` равен `local`, `user`, `open_source` или `imagegen`;
- `path` ведет к существующему `.png`, `.jpg`, `.jpeg` или `.webp`;
- `status` равен `ok`;
- `created_with` не равен `svg`, `svg_origin`, `html_css`, `canvas` или
  `screenshot`;
- `evidence` заполнен для источников `imagegen`, `open_source` и `user`.

Резервный вариант проходит проверку только при выполнении всех условий:

- `source_kind` или `status` равен `svg_fallback` или `placeholder`;
- `fallback_approved_by_user` равен `true`;
- `fallback_reason` объясняет, почему реальный PNG/JPG/WebP не использован;
- финальная сдача называет визуал резервным вариантом, а не растровым источником.

Если резервный вариант не подтвержден, остановись и запроси файлы, разрешение
на генерацию изображения, поиск источников или явное подтверждение запасного варианта. Не
строй слайд вокруг неподтвержденного резервного варианта.

## CLI-проверка

Используй локальную копию скрипта из рабочей директории презентации:

```bash
node scripts/asset_gate_check.mjs --manifest asset-manifest.json --mode predesign
node scripts/asset_gate_check.mjs --slides slides --manifest asset-manifest.json --mode preexport
```

`export_deck_png.mjs` и `export_deck_pptx.mjs` запускают эту проверку перед
экспортом. Если manifest отсутствует или невалиден, экспорт должен упасть.

## Стоп-правила агента

- Не начинай визуальный дизайн для защищённого визуала без проходящей записи в
  manifest.
- Не считай «imagegen недоступен» подтверждением запасного варианта.
- Не выполняй просьбу «без вопросов, просто нарисуй SVG» для защищённого визуала. Это
  блокер, пока пользователь явно не подтвердит запасной вариант после объяснения, что
  это не реальный растровый источник.
- Не прячь SVG-запас за расширением `.png`. Происхождение определяется через
  `source_kind`, `created_with`, `evidence` и `fallback_approved_by_user`, а не
  только через расширение файла.
