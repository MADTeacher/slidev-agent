# Процесс презентационного SVG

Используй этот файл, когда нужно создать SVG-слайд с нуля, усилить слабую композицию, сделать обложку, разделитель, декоративную сцену, схему, компактный векторный ассет или подтверждённый запасной вариант для сложного объекта после `asset-gate.md`.

## Бриф

Сними бриф коротко. Если пользователь не дал ответ, выбери безопасное значение и назови его в сдаче.

- Роль в деке: обложка, разделитель, схема, фон, иконка, выноска, финальный слайд, запасной предметный визуал.
- Сюжет или смысл: что зритель должен понять за первые две секунды.
- Формат: 16:9, 4:3, квадрат, вертикальный постер, маленький ассет.
- Стиль: строгий, мягкий, технический, редакционный, учебный, плакатный, декоративный.
- Настроение: спокойное, напряжённое, тёплое, холодное, праздничное, исследовательское, собранное.
- Палитра: токены колоды, бренд, светлая, тёмная, контрастная, пастельная, ограниченная.
- Экспорт: HTML/PDF/PNG, редактируемый PPTX, отдельный SVG-файл или вставка через `<img>`.
- Ограничения: без внешних ресурсов, без скриптов, без мелкого текста, без тяжёлых фильтров.
- Причина SVG-запаса: файл не найден, источник не подошёл, растровая генерация недоступна, результат слабый, задача не даёт использовать растр, подходящего эмоджи нет или эмоджи нечитабелен/двусмысленен. Для защищённого визуала причина не заменяет одобрение запасного варианта.

## Размеры по умолчанию

| Сценарий | `viewBox` | Детализация | Цвет |
|---|---:|---|---|
| Полный 16:9 слайд | `0 0 1920 1080` | средняя или высокая | 5-8 основных цветов |
| Компактный 16:9 ассет | `0 0 1200 675` | средняя | 4-7 основных цветов |
| Квадратная иллюстрация | `0 0 1024 1024` | средняя | 4-7 основных цветов |
| Вертикальный постер | `0 0 1080 1920` | высокая | 5-9 основных цветов |
| Маленькая иконка | `0 0 512 512` | низкая или средняя | 2-5 основных цветов |

## План сцены

Сначала опиши для себя сцену как слои. Потом пиши SVG.

1. Холст: `svg`, `viewBox`, `role="img"`, `title`, `desc`.
2. `defs`: градиенты, маски, паттерны, фильтры, символы.
3. Фон: цвет, мягкий градиент, сетка, крупная пустая зона под текст.
4. Крупные массы: главный силуэт, диагонали, рамки, смысловой центр.
5. Средний план: схемные элементы, повтор, вторичные объекты, линии связи.
6. Свет: источник, тени, полупрозрачные формы, локальный glow, отражения.
7. Фактура: тонкие линии, точки, паттерны и ритм только там, где это помогает фокусу.
8. Акценты: маленькие контрастные детали, блики, метки, выносные линии.
9. Интеграция: безопасные поля, место под заголовок, совпадение с токенами колоды.

## Композиционные решения

- Для обложки оставь место под заголовок и подзаголовок. Проверяй, что SVG не спорит с текстовым слоем.
- Для сцены держи передний, средний и дальний план. Так появляется глубина без фотореализма.
- Для схемы сначала выбери поток чтения: слева направо, сверху вниз, радиально или по кольцу.
- Для эмблемы используй крупный силуэт, внутренний знак, чистый контур и ограниченную палитру.
- Для орнамента строй модуль и повтор, затем добавь один контролируемый сбой ритма, если сцене нужна живость.

## Презентационные ограничения

Если нужен редактируемый PPTX, держи ключевой текст отдельными HTML/PPTX-блоками. SVG вставляй как визуальный слой. Внутри SVG оставляй только декоративные микроподписи или технические метки, которые допустимо растеризовать.

Для `Young ESL` и детских ESL-слайдов сначала проверь, может ли простую роль закрыть системный эмоджи нужного размера: подсказка раздела, игровой маркер, погодный сигнал, проверка, повторение или действие. Если эмоджи не найден, нечитабелен или двусмысленен, зафиксируй это как причину SVG-запаса и только потом делай SVG. Это не относится к персонажам, одежде, сценам, еде, животным и предметным карточкам.

Если слайду нужен узнаваемый реальный объект, сначала выполни `asset-gate.md` и `content-assets.md`. Локальные материалы, открытые источники и генерация PNG/JPG/WebP идут перед сложным SVG. Если эти варианты не дали пригодного результата, остановись и запроси одобрение запасного варианта. Только после этого SVG может стать самостоятельным запасным материалом: стилизованной иллюстрацией, учебной схемой, условной моделью, метафорической сценой или предметным образом.

Такой SVG не подавай как фотографию, официальный продуктовый рендер, медицински точную картинку или документальный источник. Держи формулировку честной: «стилизованная SVG-иллюстрация», «условная SVG-модель», «SVG-схема», «запасной SVG-визуал». Если нужен редактируемый PPTX, подписи, термины и учебные слова остаются отдельными HTML/PPTX-блоками.

## Цикл правки

1. Создай SVG по плану слоёв.
2. Запусти `node scripts/presentation_svg_pipeline.mjs audit <file.svg> <preview.png> --width 1920 --height 1080`.
3. Открой PNG и оцени картинку по `visual-review.md`.
4. Исправь SVG: фокус, контраст, читаемость, глубину, цвет, лишний шум.
5. Повтори цикл. Для сложной сцены обычно хватает двух проходов.

## Когда уточнять бриф

- Размер, формат сдачи или место использования меняют архитектуру файла.
- Пользователь просит точное сходство с человеком, логотипом, товарным знаком или защищённым персонажем.
- Нужен внешний ресурс, который нельзя безопасно встроить.
- Нужен стиль живого художника; замени его нейтральным описанием эпохи, техники, палитры, ритма линий и плотности деталей.
- Нужна анимация, а среда публикации может запретить SMIL или CSS-анимацию.
