# Безопасность презентационных SVG

Используй этот файл при работе с чужим SVG, веб-встраиванием, экспортом в HTML, подозрительными ссылками и финальной проверкой.

## Главный риск

SVG: XML-документ с возможностью ссылок, стилей, анимации и встраивания. В небезопасной среде он может запустить код, загрузить внешний ресурс или сломать экспорт презентации.

## Запрещённые элементы и признаки

| Признак | Почему опасно | Что делать | Проверка |
|---|---|---|---|
| `<script>` | Запускает JavaScript | Удалить полностью | Скрипт не найден валидатором |
| `foreignObject` | Может включать HTML | Заменить SVG-формами | Элемент отсутствует |
| `iframe`, `object`, `embed` | Встраивает внешний контент | Удалить | Элементы отсутствуют |
| `audio`, `video`, `canvas` | Тянет неподходящий медиа-слой | Заменить статичной графикой | Элементы отсутствуют |
| Атрибуты `onload`, `onclick`, `onerror` и похожие | Запускают события | Удалить все `on...` | Валидатор не видит обработчиков |
| `href="http..."`, `xlink:href="http..."` | Тянет сеть | Заменить внутренним `#id` или удалить | Остались только внутренние ссылки |
| `data:` | Может спрятать код или тяжёлый файл | Удалить или пересобрать SVG-формами | `data:` отсутствует |
| `javascript:` | Прямая попытка выполнить код | Удалить | Подстрока отсутствует |
| `@import` в стиле | Загружает внешний CSS | Удалить | В стилях нет импорта |
| `DOCTYPE` и `ENTITY` | Риск внешних сущностей | Удалить | XML без сущностей |

## Разрешённые внутренние ссылки

Можно использовать ссылки вида `href="#symbolId"`, `fill="url(#gradientId)"`, `clip-path="url(#clipId)"`, `mask="url(#maskId)"`. Они должны ссылаться на элементы внутри того же SVG.

## Ручная проверка

- Найди все `href`, `xlink:href`, `src`, `url(...)` и проверь, что они не ведут в сеть.
- Найди все атрибуты, начинающиеся с `on`, и удали их.
- Найди `script`, `foreignObject`, `iframe`, `object`, `embed`, `DOCTYPE`, `ENTITY`.
- Проверь пользовательский текст: `&`, `<`, `>`, кавычки в атрибутах должны быть экранированы.
- Убедись, что итоговый SVG открывается как картинка и не требует внешних файлов.

## Командная проверка

```bash
node scripts/presentation_svg_pipeline.mjs validate image.svg
node scripts/presentation_svg_pipeline.mjs audit image.svg preview.png --width 1920 --height 1080
```

`validate` проверяет опасные конструкции. `audit` сначала проверяет SVG, затем пытается создать PNG для визуальной оценки.

## Сдача веб-файла

Если SVG пойдёт в HTML-презентацию, отдавай безопасный самодостаточный файл. Если нужна интерактивность, вынеси её в отдельный кодовый слой и попроси пользователя подтвердить контекст публикации, политику безопасности сайта и права на внешние ресурсы.
