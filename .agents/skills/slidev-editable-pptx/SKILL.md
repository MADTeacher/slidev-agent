# Slidev Editable PPTX

Use this skill when the user wants an approved Slidev deck exported to an **editable** PowerPoint file.

This skill is project-local and must work the same way in both Codex and opencode. Do not depend on global skills or runtime-specific PowerPoint builders.

## Required Inputs

- `slides.md` in the project root
- `output/<slug>.deck-spec.json` created by the updated `slidev-assembler`
- Any referenced assets in `public/`

If the deck spec is missing, stop and report that the deck must be reassembled with the updated pipeline before editable export can run.

## Contract

- The default PPTX export path is `bun run export:pptx`.
- This script must create `output/<slug>.pptx` as an editable PowerPoint generated by repo-local code.
- The deck spec is the source of truth for editable export. Do not parse PDF, HTML, or legacy PPTX back into a new deck.
- The default export is now a **native PowerPoint build**, not a raster or hybrid overlay export.
- Approved `PNG`/`PDF` exports are the visual source of truth for fidelity checks. If the native PPTX rendering diverges from the approved deck on a slide or region, match the approved render instead of inventing a new layout.

## Required Steps

1. Validate the deck spec:
   ```bash
   bun run validate:deck-spec
   ```
2. Export the editable PPTX:
   ```bash
   bun run export:pptx
   ```
3. Verify the expected output file exists in `output/`.
4. Sanity-check the package:
   - the final archive opens cleanly as a `.pptx`
   - slide/layout XML should not retain starter-template placeholders like `p:ph`
   - editable text should remain native PowerPoint text objects, not full-slide image overlays
5. For slides with diagrams or fragile visual compositions, compare the PPTX result against the approved `PNG`/`PDF` render.
   - If the native rebuild materially diverges, keep surrounding text native and embed the approved diagram region as an image inside the slide.
   - Prefer a cropped diagram/image region from the approved slide render over a full-slide raster fallback.

## Editable Scope

Keep these elements native PowerPoint objects whenever the deck spec provides them:

- slide titles and subtitles
- bullets, body copy, captions, callouts
- speaker notes
- stat cards
- tables
- comparison blocks
- simple process and architecture blocks
- charts with structured data

These may remain images when needed:

- complex SVG illustrations
- Mermaid diagrams that are too expensive to rebuild 1:1
- decorative background art

In the current native PPTX pipeline, complex diagram slides may be embedded as PNG media inside the deck while surrounding titles, bullets, and notes remain native PowerPoint content.

For Mermaid-heavy or layout-sensitive slides, use this decision rule:

- First try a native PowerPoint reconstruction only if it stays visually close to the approved render.
- If the reconstruction drifts in structure, spacing, routing, or readability, stop forcing native shapes.
- Use the approved exported slide PNG as the fidelity reference and crop/embed only the diagram region needed in PPTX.
- Preserve native editability for adjacent text blocks, labels, bullets, and notes whenever possible.

If a visual remains an image, adjacent meaning-bearing text must still stay editable.

## Implementation Notes

- Use only repo-local scripts and dependencies from `package.json`.
- The default builder is `scripts/native-pptx/build-deck-native.mjs`.
- Deck-specific native overrides belong in `scripts/native-pptx/deck-configs/<slug>.config.mjs`, not in a new per-deck builder file.
- The native export pipeline now includes post-processing cleanup of the generated package to remove inherited PowerPoint template placeholders and other starter-layout artifacts.
- The preferred fallback for hard-to-port diagrams is: approved slide render -> crop the diagram region -> embed that region as PNG media in the native PPTX.
- The deck spec schema is `schemas/deck-spec.schema.json`.
- Asset references must resolve against `public/` unless an absolute path is explicitly provided.
- Verification artifacts for the native builder are written under `tmp/slides/<deck-id>/`.

## Output

Report the editable PPTX path, whether the export completed successfully, and whether placeholder cleanup passed.
